If you have questions, encounter a bug, or want to discuss ideas for the Worlds Platform, use the channels below based on your need.

## Before you ask

The fastest path to an answer is often self-service. Check these resources first:

<Steps>
  <Step title="Search the documentation">
    Use the search bar at the top of this site to find relevant pages. The [Quickstart](/overview/getting-started/quickstart) and [Developer guide](/contributing/developer-guide) cover the most common setup questions.
  </Step>

  <Step title="Search GitHub Issues">
    Browse [open and closed issues](https://github.com/wazootech/worlds/issues) on GitHub. Your question may already have an answer, or a known workaround.
  </Step>

  <Step title="Check existing discussions">
    Review [GitHub Discussions](https://github.com/wazootech/worlds/discussions) for community Q&A and design conversations.
  </Step>
</Steps>

## Reporting bugs

When opening a bug report on GitHub, include:

- **A clear title** — summarize the problem in one line
- **Steps to reproduce** — the shortest sequence of commands or code that triggers the issue
- **Expected behavior** — what you expected to happen
- **Actual behavior** — what actually happened, including any error messages or stack traces
- **Environment** — Deno version (`deno --version`), OS, and relevant package versions

<Note>
  For security vulnerabilities, do not open a public GitHub issue. Contact the team directly at [support@wazoo.dev](mailto:support@wazoo.dev).
</Note>

## Asking questions

GitHub Discussions is the best place for open-ended questions. When asking:

- Describe what you are trying to achieve, not just what is failing
- Include relevant code snippets (with language tags)
- Share what you have already tried
- Provide your environment details if the question is runtime-specific
